## Audience Tabs

### 🧒 Simple

**What is cloud computing?**

The "cloud" is just other people's computers — specifically, large data centres owned by companies like Amazon, Microsoft, or Google, rented out over the internet.

When you use Google Drive, Dropbox, or a business app — your files and data are stored on those remote computers, not just on your device.

**Why it matters for you:**

* Businesses use cloud to run websites, store data, and run software
* Consumers use cloud every time they use Gmail, WhatsApp backups, or iCloud
* Where your data is stored affects your privacy and legal rights

**For a small business or website:**

* **Shared hosting** — cheapest; your website shares a server with others
* **VPS** — a virtual dedicated server; more power and control
* **Cloud hosting** — flexible; scales with demand
* **Managed hosting** — someone else handles the technical setup for you

### 🧑 Detailed

## Overview

Cloud computing and web hosting are related but distinct concepts that affect anyone running a website, application, or digital service in Malaysia.

**Web hosting** provides the infrastructure for a website or application to be accessible online. **Cloud computing** is broader — covering compute, storage, databases, networking, AI services, and more via remote infrastructure.

For Malaysian businesses, the key decisions involve: which provider to use, where data is physically stored, and what compliance obligations apply.

## Part 1: What Cloud Computing Is

### The Three Cloud Service Models

| Model | What It Provides | You Manage | Examples |
| --- | --- | --- | --- |
| IaaS (Infrastructure as a Service) | Virtual machines, storage, networking | Operating system, applications, data | AWS EC2, Azure VM, Google Compute Engine |
| PaaS (Platform as a Service) | Runtime environment, databases, middleware | Applications and data | Google App Engine, AWS Elastic Beanstalk, Heroku |
| SaaS (Software as a Service) | Complete application | Nothing — just use it | Google Workspace, Microsoft 365, Salesforce |

### The Three Deployment Models

| Deployment | Description | Best For |
| --- | --- | --- |
| Public cloud | Shared infrastructure managed by provider | Most businesses — cost-efficient, scalable |
| Private cloud | Dedicated infrastructure for one organisation | Regulated industries with data sensitivity requirements |
| Hybrid cloud | Mix of public and private | Enterprises with some sensitive workloads |

### Why Malaysia's Cloud Regions Matter

Having a cloud region in Malaysia means data can be processed and stored in-country — relevant for:

* **Latency** — lower ping for Malaysian users
* **Data sovereignty** — data stays in Malaysia, subject to Malaysian law
* **Regulatory compliance** — some sectors (banking, healthcare, government) require local data residency

## Part 3: Web Hosting — Types Explained

### Hosting Type Comparison

| Type | Description | Monthly Cost (approx.) | Best For |
| --- | --- | --- | --- |
| Shared hosting | Your site shares server resources with many others | RM 5–RM 30 | Personal blogs, small business sites with low traffic |
| VPS (Virtual Private Server) | Dedicated virtual resources on a shared physical server | RM 30–RM 200 | Growing SMEs, developers, medium-traffic sites |
| Dedicated server | Physical server entirely for you | RM 300–RM 2,000+ | High-traffic sites, sensitive data, large applications |
| Managed WordPress | Shared or VPS optimised and managed for WordPress | RM 20–RM 100 | WordPress sites without in-house technical staff |
| Cloud hosting | Resources scale automatically with demand | Variable (pay-per-use) | E-commerce, apps with variable traffic |
| Managed cloud | Cloud + technical management handled by provider | RM 200–RM 2,000+ | Businesses without DevOps capability |

### What "Bandwidth" and "Storage" Mean

| Term | What It Means Practically |
| --- | --- |
| Storage (GB/TB) | How much data your website files, databases, and email can occupy |
| Bandwidth | How much data can be transferred per month — important for high-traffic sites |
| Uptime SLA | Percentage guarantee that the server is online (99.9% = ~8.7 hours downtime/year) |
| SSL certificate | Enables HTTPS — essential for security and Google ranking; most hosts include free Let's Encrypt |
| cPanel / Plesk | Control panel for managing hosting — files, email, databases without command line |

## Part 4: How to Choose — By Use Case

### Personal Website or Portfolio

**Recommended:** Shared hosting from a local provider (ExaBytes, Shinjiru, ServerFreak)

* Cost: RM 5–RM 20/month
* What you get: Domain + hosting + email + WordPress installer
* Technical requirement: Low — cPanel-based management

### Small Business Website (Brochure / Informational)

**Recommended:** Managed WordPress or shared hosting

* Cost: RM 20–RM 50/month
* Key requirements: SSL certificate, uptime reliability, Malaysian support
* Local providers (ExaBytes, ServerFreak) offer BM-language support — useful for non-technical owners

### E-Commerce Store (SME)

**Recommended:** Managed WooCommerce or Shopify (SaaS) or VPS

* Cost: RM 80–RM 300/month
* Key requirements: PCI DSS compliance for payments; good uptime; performance under peak traffic (11.11 sales)
* Consider: Shopify (SaaS, managed) vs WooCommerce on VPS (more control, more technical)

### Web Application or API (Startup / Developer)

**Recommended:** AWS, GCP, or Azure (Malaysian region preferred for data residency)

* Cost: Pay-as-you-go; can start very cheaply
* Key requirements: Scalability; managed databases; CI/CD pipeline support; Malaysian IP / low latency
* Tools: AWS Amplify, Google Firebase, Azure App Service for serverless/managed stacks

### Enterprise or Government

**Recommended:** Azure Malaysia, AWS Malaysia, TM ONE Cloud, or AIMS colocation

* Key requirements: Data residency in Malaysia; security certifications (ISO 27001, CSM); SLA; compliance with PDPA and sector regulations
* Government cloud: TM ONE has specific government cloud offerings aligned with Malaysian public sector requirements

## Part 5: Data Sovereignty and Compliance in Malaysia

### PDPA and Cloud Storage

The **Personal Data Protection Act 2010 (PDPA)** restricts transfer of personal data outside Malaysia unless:

* The destination country has adequate data protection (Minister-approved list)
* The data subject has consented to the transfer
* The transfer is necessary for a contract

**Practical implication for businesses using cloud:**

Using AWS Singapore, GCP Singapore, or any cloud region outside Malaysia technically involves transferring data out of Malaysia — and may require consent or other compliance steps under PDPA.

Using Malaysian cloud regions (AWS Malaysia Local Zone, Azure Malaysia) keeps data in-country and simplifies PDPA compliance.

### Sector-Specific Requirements

Certain Malaysian sectors have explicit data localisation or security requirements:

| Sector | Requirement | Relevant Regulation |
| --- | --- | --- |
| Banking and finance | Core banking data must remain in Malaysia | BNM Risk Management in Technology (RMiT) Policy |
| Healthcare | Patient records subject to confidentiality obligations | Private Healthcare Facilities and Services Act |
| Government | Sensitive government data on government-approved cloud | MAMPU cloud policy |
| Telecommunications | Customer data subject to MCMC obligations | Communications and Multimedia Act |

### Bank Negara RMiT Policy (Financial Sector)

BNM's **Risk Management in Technology (RMiT)** policy requires:

* Critical systems and data to remain in Malaysia (or with BNM approval for offshore)
* Cloud providers to meet BNM's security and audit requirements
* Third-party cloud audits for financial institutions

This is why Malaysian banks use domestic cloud regions (Azure Malaysia, AWS Malaysia) or TM ONE for core systems.

### The Malaysian Government Cloud Programme

MAMPU (Malaysia Administrative Modernisation and Management Planning Unit) oversees the government cloud strategy:

* Approved cloud providers for government use must meet Malaysian government security standards
* TM ONE, AWS, Azure, and selected providers are on the approved list
* Sovereign cloud options are increasingly offered by major providers for government use

## Practical Habits for Cloud and Hosting Decisions

Businesses and developers who manage cloud infrastructure well tend to:

* Use Malaysian cloud regions wherever possible for PDPA compliance and latency
* Review cloud bills monthly — cloud costs can escalate significantly with unused resources
* Enable billing alerts and budgets on all cloud accounts
* Use managed services (RDS, managed Kubernetes) rather than self-managing where possible — reduces operational burden
* Keep backups in a different region from primary data
* Conduct annual review of hosting plans — SME needs outgrow shared hosting quickly; upgrade proactively rather than reactively

## Why This Matters

**Scenario 1: Data stored overseas, PDPA compliance issue**

A Malaysian HR software company stored all employee personal data (IC numbers, salaries, addresses) on an AWS Singapore instance — not realising this constituted an overseas data transfer under PDPA. When an enterprise client's legal team raised the question during due diligence, the company had to migrate to AWS Malaysia Local Zone, implement consent processes, and update their privacy policy. Planning for data residency from the start would have avoided this.

**Scenario 2: SME outgrew shared hosting mid-campaign**

An online store using RM 20/month shared hosting ran an 11.11 sale campaign. The traffic spike during the promotion caused the server to time out — the shared hosting plan had resource caps. Hundreds of orders were lost during the outage window. Upgrading to VPS or cloud hosting before a high-traffic campaign would have cost RM 60–80/month and prevented the downtime.

**Scenario 3: Local provider, local support**

A small Malaysian F&B business set up their website on a foreign hosting provider with English-only support. When they encountered a technical issue, the time zone difference and English-only chat meant two days of downtime. A local Malaysian provider (ExaBytes, ServerFreak) with BM-language phone support would have resolved the issue same-day.

## FAQ

**Q: Is using Google Drive or Microsoft OneDrive a PDPA compliance issue for businesses?**
Potentially, if storing personal data of Malaysian individuals on overseas servers without consent or other PDPA-compliant basis. Microsoft 365 (OneDrive) with Azure Malaysia data residency settings enabled can keep data in Malaysia. Google Workspace offers regional data storage controls. Check your provider's data residency settings before using SaaS tools for personal data.

**Q: What is the difference between a data centre and a cloud region?**
A data centre is a physical facility containing servers. A cloud region is a geographic area where a cloud provider has one or more data centres — offering their services from that location. AWS, Azure, and Google all operate regions within Malaysia, meaning your data can be processed locally.

**Q: Is Huawei Cloud safe to use for business data?**
Huawei Cloud is technically capable and competitively priced. The consideration is geopolitical — some enterprises and regulated industries are cautious about using Chinese-owned cloud infrastructure for sensitive data due to data access concerns under Chinese law. Assess based on your data sensitivity, regulatory environment, and risk tolerance.

**Q: What is a CDN and do I need one?**
A CDN (Content Delivery Network) distributes your website's static content across servers worldwide, so visitors load content from a nearby server — reducing load time. Cloudflare offers a free CDN and also provides DDoS protection and SSL. For any website with Malaysian or regional users, Cloudflare's free tier is a no-brainer addition.

**Q: Do I need a Malaysian domain (.my) for my business website?**
Not legally required — but a .my domain signals local presence and builds trust with Malaysian customers. MyNIC (the Malaysian Network Information Centre) manages .my domains — register via accredited registrars including Shinjiru, ExaBytes, and others. .my domains require proof of Malaysian entity (IC for individuals, SSM for businesses).
